![logo](./images/logo.png)

**SunAllocator** — кастомна інтеграція для Home Assistant, яка допомагає максимально використовувати енергію сонячних панелей. Вона розраховує надлишкову потужність і автоматично розподіляє її між вашими пристроями — бойлером, зарядкою електромобіля тощо, перетворюючи ваш будинок на «сонячного вампіра», що споживає всю вільну зелену енергію.

## Можливості

- Розрахунок вільної (надлишкової) потужності сонячних панелей та відсотку їх використання.
- Оцінка максимально можливої потужності при поточній напрузі на основі алгоритму MPPT.
- **Три методи розрахунку надлишку на вибір** — `mppt` (обережний, типовий), `mppt_probe` (активне зондування з валідацією батареєю для відновлення куртейлд-енергії) та `export` (енергобаланс для інверторів з експортом у мережу). Вибір у розширених налаштуваннях.
- Автоматичне керування пристроями за пріоритетом (перемикачі, освітлення, кліматичні пристрої, ESPHome-реле).
- Підтримка режимів вкл/викл та пропорційного (диммерного) керування.
- Налаштовуваний дебаунс, гістерезис та мінімальний час роботи для захисту техніки від частого перемикання.
- Температурна компенсація для точного розрахунку потужності панелей.
- Розклад: за часом або через хелпер-сутність HA (наприклад `input_boolean`, schedule helper).
- Кліматичні пристрої: автоматичний підбір `hvac_mode` зі списку підтримуваних (`heat` → `heat_cool` → `auto`).
- Стартовий захисний період: пристрій не вимикається одразу після ввімкнення.
- **Сенсори та світч авто-контролю на кожен пристрій** (див. нижче).
- **Підтримка Multi-MPPT** — налаштуйте до 4 незалежних MPPT-трекерів для точного розрахунку потужності складних сонячних масивів.
- **Світч паузи авто-контролю** на пристрій — миттєво вимикає авто-керування без видалення з конфігу.
- **Вимкнення при відключенні авто-контролю** — опційно надсилає команду вимкнення пристрою при відключенні його авто-керування.
- **Блокування запуску за рівнем заряду батареї (SOC)** — опційно блокує нові запуски пристроїв поки батарея не зарядиться до заданого рівня (% на пристрій, з гістерезисом для запобігання частому перемиканню).
- **Пріоритет заряду батареї** (`battery_sharing_soc`) — нижче заданого порогу SOC батарея забирає весь заряд; вище — застосовується ватний резерв і надлишок розподіляється між пристроями. 0 = вимкнено.
- **Активне зондування** (`mppt_probe`) — коли батарея на межі заряду й інвертор душить панелі (MPPT-оцінка занижує реальний потенціал), плавно нарощує кероване навантаження й валідує батареєю, відновлюючи інакше змарноване сонце. Вимикається на пристрій (`allow_probe`).
- **Прогноз генерації PV** (опційно) — підключіть зовнішній сенсор прогнозу (напр. Forecast.Solar / Open-Meteo); показується як діагностика (`forecast_potential_w`, `forecast_untapped_w`) і, якщо заданий, слугує ціллю зондування з валідацією батареєю. Опублікований надлишок завжди лишається обережним.
- **Виявлення куртейлменту** — діагностичний прапорець `curtailment_detected`, що інвертор душить панелі.
- **Сенсор реальної потужності на пристрій** — передає реальне споживання пристрою алокатору для точнішого розрахунку залишкового бюджету; нижче порогу `idle` пристрій повідомляє `idle` замість `active`.
- **Макс. час роботи на добу** — обмежує сумарний добовий час роботи пристрою; після вичерпання він вимикається і блокується.
- **Шаблон умови придатності** — гейтить пристрій довільним Jinja-шаблоном (наприклад, температура бака) понад його розклад.
- **Режим симуляції** (лише debug) — увімкніть рівень `DEBUG` для логера `custom_components.sun_allocator`, щоб відкрити прихований тестовий режим, який підставляє фіксовані показники PV замість реальних сенсорів — для перевірки логіки розподілу без сонця.
- **Повне налаштування через інтерфейс Home Assistant** — без редагування YAML.

## Встановлення

1.  Якщо ще не встановлено — встановіть [HACS](https://hacs.xyz/).
2.  Перейдіть до HACS > Інтеграції > додайте репозиторій через «Кастомний репозиторій».
3.  Знайдіть «SunAllocator» і встановіть.
4.  Перезапустіть Home Assistant.
5.  Перейдіть до **Налаштування → Пристрої та служби**.
6.  Натисніть **«+ ДОДАТИ ІНТЕГРАЦІЮ»**.
7.  Знайдіть **«SunAllocator»** і виберіть її.
8.  Пройдіть майстер налаштування для конфігурації панелей і пристроїв.

Також можна вручну скопіювати папку `custom_components/sun_allocator` до директорії `config/custom_components` вашого Home Assistant.

## Базове налаштування

SunAllocator налаштовується виключно через інтерфейс HA. Майстер налаштування проведе вас через два основні кроки:

1.  **Налаштування сонячних панелей** — вкажіть сенсори потужності та напруги, а також параметри з datasheet панелі (Vmp, Imp тощо).
2.  **Налаштування пристроїв** — додайте перемикачі, освітлення або ESPHome-реле для керування надлишковою сонячною потужністю. Для кожного пристрою можна задати пріоритет.

Детальний опис усіх параметрів: [**Детальна конфігурація**](./docs/configuration_uk.md).

## Використання

### Сенсори рівня хабу

Інтеграція створює кілька сенсорів для моніторингу сонячної системи:

-   `sensor.sun_allocator_excess_power` — доступна надлишкова потужність. Використовуйте для тригерів автоматизацій.
-   `sensor.sun_allocator_current_max_power` — оцінена максимальна потужність панелей при поточній напрузі.
-   `sensor.sun_allocator_usage_percent` — поточне навантаження у відсотках від максимально можливої потужності.
-   `sensor.sun_allocator_power_distribution` — загальна потужність, розподілена між усіма керованими пристроями, плюс діагностичні атрибути на кожен пристрій (`allocation_w`, `allocation_percent`, `device_meta`, `reasons`).

### Сутності на кожен пристрій

Для кожного налаштованого пристрою додатково створюються:

-   `sensor.sun_allocator_<device_name>_power` — поточна виділена потужність у Вт.
-   `sensor.sun_allocator_<device_name>_power_percent` — пропорційне навантаження у %.
-   `sensor.sun_allocator_<device_name>_device_status` — ENUM-сенсор зі станами: `active`, `insufficient_power`, `debouncing_on`, `debouncing_off`, `auto_control_off`, `manual_override`, `filtered`, `trying_on`, `trying_off`, `failed_on`.
-   `switch.sun_allocator_<device_name>_auto_control` — runtime-світч авто-керування пристроєм. Стан переживає перезапуск Home Assistant (`RestoreEntity` + синхронізація з конфігом). Вимикання миттєво зупиняє авто-керування без видалення пристрою з конфігу.

### Приклади автоматизацій

Вмикає перемикач при надлишку понад 50 Вт:

```yaml
automation:
  - alias: "Увімкнути пристрій при надлишку сонячної енергії"
    trigger:
      - platform: numeric_state
        entity_id: sensor.sun_allocator_excess_power
        above: 50
    action:
      - service: switch.turn_on
        target:
          entity_id: switch.your_device
```

Вимикає авто-керування бойлером уночі (через runtime-світч):

```yaml
automation:
  - alias: "Пауза авто-керування бойлером на ніч"
    trigger:
      - platform: time
        at: "22:00:00"
    action:
      - service: switch.turn_off
        target:
          entity_id: switch.sun_allocator_heater_auto_control
  - alias: "Відновити авто-керування бойлером зранку"
    trigger:
      - platform: time
        at: "07:00:00"
    action:
      - service: switch.turn_on
        target:
          entity_id: switch.sun_allocator_heater_auto_control
```
