# Технічні концепції

Цей документ пояснює основні концепції та розрахунки, що використовуються компонентом SunAllocator.

## Розрахунок надлишкової потужності

Сенсор `sun_allocator_excess_power` розраховує невикористаний потенціал потужності, яку можна отримати від сонячної панелі. Розраховується за формулою:

```
надлишок_потужності = розрахункова_макс_потужність_при_поточній_напрузі - поточна_вихідна_потужність_ФЕМ
```

Це значення показує, скільки додаткових ватів можна отримати від панелі при її поточній робочій напрузі. Коли це значення велике, це свідчить про те, що панель має значний невикористаний потенціал, який можна задіяти за допомогою додаткових навантажень.

## Розуміння MPPT та роботи сонячних панелей

Сонячні панелі мають характеристичну криву I-V (струм-напруга), яка визначає їхню вихідну потужність. Ключові моменти для розуміння:

1.  **Точка максимальної потужності (MPP)**: Це оптимальна робоча точка, де добуток напруги та струму є максимальним. Вона відповідає конкретним значенням напруги (Vmp) та струму (Imp).

2.  **Робочі зони**:
    *   **Нижче Vmp**: У цій зоні панель працює зі відносно постійним струмом. Зі збільшенням напруги потужність зростає приблизно лінійно.
    *   **При Vmp**: Це оптимальна робоча точка з максимальною вихідною потужністю.
    *   **Вище Vmp**: У цій зоні струм різко падає зі збільшенням напруги, що призводить до зниження потужності.

3.  **Індикатор можливого надлишку**: Атрибут `excess_possible` має значення `true`, коли напруга панелі перевищує Vmp. Це свідчить про:
    *   Панель працює в зоні постійної напруги (вище Vmp).
    *   Вихідна потужність, ймовірно, менша за оптимальну.
    *   Контролер MPPT потенційно може збільшити потужність, знизивши напругу до Vmp.

4.  **Чому зазвичай «false»**: Більшість сонячних систем з контролерами MPPT працюють при напрузі Vmp або нижче для максимізації потужності, тому `excess_possible` зазвичай має значення «false» під час нормальної роботи.

## Розрахунок максимальної потужності на основі MPPT

Значення `current_max_power` оцінює максимальну потужність, яку можна отримати при поточній напрузі, на основі принципів відстеження точки максимальної потужності (MPPT). Це допомагає зрозуміти:

1.  Яку потужність ваша панель потенційно може виробляти при поточній робочій напрузі.
2.  Чи працюєте ви поблизу оптимальної напруги для максимальної потужності.
3.  Яку додаткову потужність можна отримати при оптимальному керуванні MPPT.

#### Технічні деталі алгоритму MPPT

SunAllocator використовує вдосконалену модель для оцінки максимально можливої потужності при будь-якій напрузі:

1.  **Розрахунок відносної напруги**:
    ```
    # Послідовне з'єднання: напруги панелей підсумовуються
    відносна_напруга = виміряна_напруга_ФЕМ_В / (номінальна_Vmp_панелі_В * кількість_панелей_послідовно)

    # Паралельне з'єднання: напруга однакова для всіх панелей
    відносна_напруга = виміряна_напруга_ФЕМ_В / номінальна_Vmp_панелі_В
    ```
    Значення `1.0` означає, що масив працює точно в точці максимальної потужності (Vmp).

2.  **Модель оцінки потужності**:
    *   Для напруги нижче або рівної Vmp (`відносна_напруга ≤ 1.0`):
        ```
        # Поліноміальне наближення: плавний підйом до Vmp,
        # відповідає зоні постійного струму реальної кривої I-V.
        оцінкова_макс_потужність_Вт = номінальна_Pmax_панелі_Вт * (
            відносна_напруга * (1.0 - 0.1 * (1.0 - відносна_напруга))
        )
        ```

    *   Для напруги вище Vmp (`відносна_напруга > 1.0`):
        ```
        # Квадратичне падіння: потужність різко знижується після Vmp,
        # відображає стрімке зменшення струму в зоні постійної напруги.
        оцінкова_макс_потужність_Вт = номінальна_Pmax_панелі_Вт * max(
            0,
            1 - 1.5 * (відносна_напруга - 1) ** 2
        )
        ```

3.  **Конфігурація панелей**:
    Алгоритм враховує різні схеми з'єднання панелей:
    *   **Послідовне (Series)**: Напруга підсумовується, струм залишається незмінним.
    *   **Паралельне (Parallel)**: Струм підсумовується, напруга залишається незмінною.

## Алгоритм розподілу потужності

Цей розділ описує, як SunAllocator вирішує, які пристрої вмикати та скільки потужності виділяти кожному.

### Крок 1 — Розрахунок надлишкової потужності

Щоразу, коли сенсор потужності ФЕМ оновлюється, SunAllocator розраховує, скільки потужності доступно для розподілу:

```
# Режим MPPT (без сенсора споживання будинку — за замовчуванням):
надлишок_потужності_Вт = оцінкова_макс_потужність_при_поточній_напрузі_Вт
                       - власне_споживання_інвертора_Вт   # з налаштувань інтеграції
                       - зарезервована_потужність_АКБ_Вт  # з налаштувань інтеграції
```

У **Паралельному режимі** (якщо налаштовано сенсор споживання будинку), надлишок розраховується як:

```
# Паралельний режим — пряме вимірювання реально "вільної" потужності:
надлишок_потужності_Вт = вихідна_потужність_ФЕМ_Вт
                       - загальне_споживання_будинку_Вт   # все що споживає будинок зараз
                       - потужність_заряду_АКБ_Вт         # позитивне = заряд, негативне = розряд
                       - власне_споживання_інвертора_Вт   # холостий хід інвертора
```

Від'ємний надлишок означає, що будинок споживає з мережі; жодні пристрої не вмикаються.

### Крок 2 — Фільтрація пристроїв

Перед тим як пристрій буде розглянуто для розподілу, він повинен пройти кілька перевірок:

- **Автокерування увімкнено**: Обробляються лише пристрої з увімкненим автокеруванням.
- **Сутність існує**: Керована сутність HA повинна бути доступна в Home Assistant.
- **Перевірка розкладу**: Якщо у пристрою є розклад, поточний час повинен знаходитися в дозволеному вікні. Якщо пристрій наразі увімкнено, але поза розкладом, він негайно вимикається.
- **Ручне керування**: Якщо режим пристрою було вручну встановлено на `Off` або `On`, розподільник поважає це і пропускає автоматичне керування.

Пристрої, що не пройшли будь-яку перевірку, пропускаються, а причина фільтрації зберігається для діагностики.

### Крок 3 — Визначення стану пристрою (рішення увімкн./вимкн.)

Для кожного придатного пристрою розподільник вирішує, чи повинен він бути активним, використовуючи порогові значення гістерезису:

```
# Поріг вмикання: пристрій активується, коли надлишку достатньо для його мінімального споживання
поріг_вмикання_Вт  = мінімальне_очікуване_споживання_Вт     # напр. 500 Вт для тепл. підлоги

# Поріг вимикання: пристрій залишається увімкненим, поки потужність не впаде нижче цього рівня
# (зазор гістерезису запобігає мерехтінню при коливаннях сонячної генерації)
поріг_вимикання_Вт = мінімальне_очікуване_споживання_Вт - глобальний_гістерезис_Вт  # напр. 500 - 40 = 460 Вт
```

- **Увімкнути**: якщо `надлишок_Вт >= поріг_вмикання_Вт` І пристрій раніше був вимкнений.
- **Залишити увімкненим**: якщо `надлишок_Вт >= поріг_вимикання_Вт` І пристрій раніше був увімкнений.
- **Вимкнути**: якщо `надлишок_Вт < поріг_вимикання_Вт`.

Це запобігає швидкому циклічному перемиканню увімкн./вимкн., коли сонячна потужність коливається навколо порогового значення.

**Антидребезг**: Зміна стану не застосовується негайно. Пристрій переходить у стан-кандидат і змінює стан лише після того, як налаштований час антидребезгу минув без зміни стану-кандидата.

### Крок 4 — Мінімальний час роботи та стартовий захисний період

Для захисту приладів (компресорів, теплових насосів, насосів) від частого перемикання:

- **Мінімальний час роботи**: Після ввімкнення пристрій залишається ввімкненим щонайменше `min_on_time` секунд, навіть якщо потужність падає нижче порогу вимкнення.
- **Стартовий захисний період**: Коли пристрій вперше вмикається, він отримує додатковий захисний часовий проміжок в 90 секунд для того аби MPPT встиг отримати найвищу точку напруги (в цей час можливе часткове споживання з акумулятору та/або мережі).

### Крок 5 — Розподіл потужності

Пристрої сортуються за **пріоритетом** (спочатку найвищий). Розподільник перебирає їх і призначає потужність із залишкового бюджету:

#### Стандартні пристрої (увімкн./вимкн.)

Кожному активному стандартному пристрою виділяється рівно `min_expected_w` з бюджету:

```
# Пристрій отримує рівно своє мінімальне номінальне споживання
виділена_потужність_Вт      = мінімальне_очікуване_споживання_Вт  # напр. 500 Вт
залишковий_сонячний_бюджет -= виділена_потужність_Вт
```

Якщо залишкової потужності недостатньо для пристрою, він вимикається.

#### Кастомні пристрої (пропорційні)

Пропорційна ціль розраховується на основі доступної потужності та максимальної ємності пристрою:

```
# Яку частку потужності пристрою може забезпечити поточна сонячна генерація?
цільовий_відсоток = обмежити(
    5%,    # мінімум: не опускатись нижче 5% (уникає мерехтіння)
    90%,   # максимум: обмеження для запасу безпеки
    (залишковий_сонячний_бюджет_Вт / максимальне_очікуване_споживання_Вт) * 100%
)

# Реальна спожита потужність при цьому відсотку
фактично_спожито_Вт = min(залишковий_сонячний_бюджет_Вт, максимальне_очікуване_споживання_Вт * цільовий_відсоток / 100%)
```

**Стратегії розподілу** (налаштовуються в розширених налаштуваннях):
- **Послідовне заповнення (Fill one by one)**: Кожен пристрій (спочатку з найвищим пріоритетом) отримує стільки, скільки потребує. Залишкова потужність переходить до наступного пристрою.
- **Рівномірний розподіл (Distribute evenly)**: Доступна потужність розподіляється пропорційно між усіма активними пропорційними пристроями на основі їхнього `max_expected_w`.

Механізм рампування поступово збільшує або зменшує рівень потужності в кожному циклі на `ramp_step_%` з мертвою зоною для запобігання мікроколиванням.

### Крок 6 — Застосування стану до сутностей

Після прийняття всіх рішень розподільник викликає служби HA:

| Пристрій / Умова | Виклик служби HA |
|-|-|
| Стандартний перемикач / освітлення → увімкнути | `switch.turn_on` / `light.turn_on` |
| Кліматичний пристрій → увімкнути | `climate.set_hvac_mode` з налаштованим режимом HVAC |
| Будь-який пристрій → вимкнути | відповідний `turn_off` / `set_hvac_mode: off` |
| Пропорційний (освітлення) → встановити рівень | `light.turn_on` з параметром `brightness` |

Розподільник перевіряє **фактичний стан сутності HA** перед відправкою команди. Якщо сутність вже знаходиться в бажаному стані, виклик пропускається для уникнення зайвого трафіку.

### Крок 7 — Оновлення сенсорів

Після обробки розподільник зберігає результати в спільній пам'яті та надсилає сигнал диспетчера. Сенсор `sun_allocator_power_distribution` отримує цей сигнал і оновлює свій стан та атрибути поточними даними розподілу для всіх пристроїв.

## Сторожовий таймер (Watchdog)

Сторожовий таймер відстежує, чи надходять оновлення від сенсора потужності ФЕМ. Якщо протягом налаштованого тайм-ауту оновлень не надходить:

1. Всі керовані пристрої вимикаються (захисне відключення).
2. Попередження записується до журналу.
3. Автокерування автоматично відновлюється, як тільки сенсор починає знову надавати дані.
