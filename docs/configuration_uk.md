# Детальна конфігурація

SunAllocator налаштовується через інтерфейс Home Assistant. Цей документ містить детальний опис усіх параметрів конфігурації.

Після початкового налаштування змінити конфігурацію можна через **Налаштування → Пристрої та служби**, знайдіть інтеграцію **SunAllocator** і натисніть кнопку **«НАЛАШТУВАТИ»**.

Відкриється меню з такими пунктами:
- **Налаштування** — зміна основних параметрів сонячних панелей та акумулятора.
- **Керування пристроями** — додавання, редагування або видалення пристроїв.
- **Температурна компенсація** — налаштування поправок на температуру для розрахунку потужності панелей.
- **Розширені налаштування** — тонке налаштування алгоритму розподілу потужності.

## Налаштування

Цей розділ охоплює основні сенсори та фізичні характеристики вашої сонячної системи.

### Основні сенсори
- **Сенсор потужності PV** *(обов'язковий)* — сенсор, що вимірює потужність сонячних панелей у Ватах (Вт).
- **Сенсор напруги PV** *(обов'язковий для режиму MPPT)* — сенсор, що вимірює напругу панелей у Вольтах (В). Потрібен, якщо не використовується сенсор споживання будинку.
- **Сенсор струму PV** *(необов'язковий)* — сенсор струму PV в Амперах (А). Використовується для діагностики та перевірки роботи MPPT.
- **Сенсор споживання** *(необов'язковий)* — сенсор загального споживання будинку у Вт. Якщо вказано — інтеграція працює в **паралельному режимі**; якщо ні — в **режимі MPPT**.
- **Сенсор потужності акумулятора** *(необов'язковий)* — сенсор потужності акумулятора у Вт. Використовується для визначення режиму (заряд/розряд).
- **Сенсор заряду батареї** *(необов'язковий)* — сенсор відсотка заряду батареї. Якщо його вказати, для окремих пристроїв можна задати мінімальний SOC, нижче якого SunAllocator не запускатиме ці навантаження.
- **Інвертована полярність акумулятора?** *(необов'язковий)* — увімкніть, якщо сенсор акумулятора показує позитивне значення при розряді і від'ємне при заряді. За замовчуванням: від'ємне — розряд, додатне — заряд.

### Кілька MPPT-контролерів
Якщо інвертор віддає окремі вимірювання PV1 і PV2, увімкніть другий MPPT-вхід та оберіть сенсори потужності й напруги PV2. Струм PV2 необов'язковий; якщо сенсора потужності PV2 немає, інтеграція може оцінити потужність PV2 як напруга * струм.

Кожен MPPT-вхід розраховується окремо зі своєю напругою, потужністю та, за потреби, власними параметрами панелей. Потім інтеграція сумує поточну максимальну потужність і невикористаний потенціал по всіх MPPT перед розрахунком надлишку. Це важливо для інверторів із двома MPPT, бо один стринг може бути нижче MPP, а другий ще мати доступний запас.

Якщо обидва MPPT використовують однакові панелі та схему підключення, поля панелей MPPT2 можна залишити зі значеннями за замовчуванням. Змінюйте їх лише коли PV2 має іншу кількість панелей, інше підключення або інший тип панелей.

### Характеристики сонячних панелей
Ці значення вказані в datasheet (паспорті) ваших панелей.

- **Vmp (напруга при максимальній потужності)** — напруга однієї панелі при максимальній потужності.
- **Imp (струм при максимальній потужності)** — струм однієї панелі при максимальній потужності.
- **Voc (напруга холостого ходу)** — максимальна напруга однієї панелі без навантаження.
- **Isc (струм короткого замикання)** — максимальний струм однієї панелі.
- **Кількість панелей** — загальна кількість сонячних панелей у масиві.
- **Конфігурація панелей** — схема з'єднання панелей:
  - **Послідовно (Series)** — панелі з'єднані послідовно. Загальна напруга — сума напруг окремих панелей, струм незмінний.
  - **Паралельно (Parallel)** — панелі з'єднані паралельно. Загальний струм — сума струмів окремих панелей, напруга незмінна.
  - **Паралельно-послідовно (Parallel-Series)** — комбінація обох варіантів.

## Керування пристроями

У цьому розділі можна додавати, редагувати або видаляти пристрої (навантаження), якими керуватиме SunAllocator.

### Параметри пристрою
- **Назва пристрою** — зрозуміла назва для ідентифікації пристрою.
- **Тип пристрою**:
  - **Стандартний (Standard)** — пристрій із простим керуванням вкл/викл. Підтримувані домени сутностей:
    - `switch`, `input_boolean` — керується через `turn_on` / `turn_off`
    - `light` — керується через `turn_on` (на повну яскравість) / `turn_off`
    - `climate` (термостат, тепловий насос) — керується через `set_hvac_mode`. При виборі entity пристрою у випадаючому списку кожен доступний режим HVAC відображається як окремий пункт (наприклад, "Термостат — нагрів", "Термостат — охолодження"). Просто оберіть потрібний режим — інтеграція зберігає його автоматично.
    - `automation`, `script` — запускається через `turn_on` / `turn_off`
  - **Кастомний ESPHome (Custom ESPHome)** — пристрій із підтримкою пропорційного керування. Поведінка залежить від типу сутності:
    - `light` — справжнє пропорційне керування: яскравість регулюється як відсоток від доступної потужності
    - `switch`, `input_boolean`, `automation`, `script` — тільки вкл/викл; інтеграція розраховує розподіл потужності внутрішньо, але сам пристрій отримує лише команду «увімкнути» або «вимкнути»
    - `climate` — аналогічно до Standard: оберіть потрібний режим HVAC зі списку entity. Інтеграція викликає `set_hvac_mode` з обраним режимом при ввімкненні та `set_hvac_mode` з `off` при вимкненні.
- **Entity пристрою** — сутність Home Assistant, що представляє пристрій.
- **Мін. очікувана потужність (Вт)** *(обов'язковий)* — мінімальна потужність споживання пристрою у робочому стані. Використовується для визначення порогу ввімкнення.
- **Макс. очікувана потужність (Вт)** *(обов'язковий для Custom-пристроїв)* — максимальна потужність при 100% навантаженні. Використовується для пропорційного керування.
- **Датчик фактичної потужності** *(необов'язковий)* — окремий датчик потужності пристрою у Вт. Для стандартних пристроїв SunAllocator і далі використовує **Мін. очікувану потужність** як поріг дозволу ввімкнення, але після ввімкнення враховує виміряну потужність замість припущення, що пристрій завжди споживає налаштований мінімум. Це корисно для термостатів, теплої підлоги, бойлерів та інших навантажень, які можуть бути дозволені, але тимчасово не споживати енергію.
- **Бінарний датчик фактичної активності** *(необов'язковий)* — `binary_sensor`, який показує, чи навантаження фізично активне саме зараз. Коли цей датчик має стан `on`, SunAllocator вважає, що пристрій споживає **Мін. очікувану потужність**. Коли стан `off`, пристрій вважається в простої та отримує `0 Вт` після завершення стартового захисного періоду. Це зручно для клапанів теплої підлоги, термостатів та подібних пристроїв, які або споживають свою звичну фіксовану потужність, або не споживають нічого.
- **Поріг фактичної потужності (Вт)** — рівень шуму для датчика фактичної потужності. Значення на цьому рівні або нижче вважаються простоєм без виміряного навантаження. Під час стартового захисного періоду SunAllocator продовжує резервувати **Мін. очікувану потужність**, доки датчик не покаже реальне споживання або поки період не завершиться.
- **Мінімальний заряд батареї (%)** *(необов'язковий)* — не дозволяє запускати цей пристрій, коли заряд батареї нижчий за вказаний поріг. Це обмеження блокує лише новий старт і не вимикає примусово пристрій, який уже працює.
- **Пріоритет** — число від 1 до 100, що визначає порядок ввімкнення. Вищий пріоритет — першим отримує потужність.
- **Час дебаунсу (с)** — час очікування перед зміною стану пристрою. Запобігає швидким перемиканням при коливаннях потужності.
- **Мінімальний час роботи (с)** — мінімальний час у секундах, протягом якого пристрій залишається ввімкненим після старту. Корисно для компресорів і насосів. При ввімкненні також застосовується **стартовий захисний період** (налаштовується в «Розширених налаштуваннях»), протягом якого пристрій не буде вимкнений навіть при падінні потужності.
- **Вимикати пристрій при вимкненні автокерування** *(необов'язковий)* — якщо увімкнено, вимкнення перемикача auto-control одразу надішле команду `off` у відповідну сутність Home Assistant. Це обходить мінімальний час роботи та стартовий захисний період, бо вважається явною дією користувача.
- **Автоматичне керування** — увімкнення або вимкнення автоматичного керування цим пристроєм.
- **Увімкнути розклад** — увімкнення або вимкнення розкладу для цього пристрою.

### Налаштування розкладу
Поле **Режим розкладу** визначає, як обчислюється дозволене вікно керування пристроєм:
- **Вимкнено** — керування дозволене у будь-який час (за замовчуванням).
- **Стандартний** — вбудоване часове вікно з вибором днів тижня.
- **Хелпер** — керування дозволене лише коли стан вибраної хелпер-сутності HA = `on` (підходить `input_boolean`, `schedule`, або будь-яка сутність зі станами `on`/`off`).

Якщо вибрано **Стандартний**:
- **Час початку** / **Час завершення** — часове вікно. Вікна через північ (end < start) підтримуються.
- **Дні тижня** — має бути вибрано хоча б один день, інакше пристрій вважається поза розкладом.

Якщо вибрано **Хелпер**:
- **Хелпер-сутність** — будь-яка сутність зі станами `on`/`off`. Авто-керування призупиняється, коли хелпер `off`.

Примітка: розклад визначає, *коли авто-керування дозволене*, тоді як **Світч авто-керування** на пристрій (`switch.sun_allocator_<device>_auto_control`) — це runtime-перемикач, який можна змінювати з автоматизацій або дашбордів. Для роботи пристрою обидва мають дозволяти керування.

### Сутності на кожен пристрій

Коли пристрій додано, інтеграція створює для нього такі сутності:

| Сутність | Призначення |
|---|---|
| `sensor.sun_allocator_<device>_power` | Виділена потужність у Вт. |
| `sensor.sun_allocator_<device>_power_percent` | Пропорційне навантаження у %. |
| `sensor.sun_allocator_<device>_device_status` | ENUM-стан (`active`, `idle`, `insufficient_power`, `debouncing_on`/`off`, `auto_control_off`, `manual_override`, `filtered`, `trying_on`/`off`, `failed_on`). |
| `switch.sun_allocator_<device>_auto_control` | Runtime-світч авто-керування. Стан переживає перезапуск. |

Unique ID слідує патерну `<entry_id>_<device_id>_<suffix>` і стабільний між перезавантаженнями.

Коли налаштовано `Actual Power Sensor`, `Active Feedback Binary Sensor` або `Minimum Battery SOC (%)`, сенсори потужності та статусу пристрою також показують додаткову діагностику на кшталт `actual_power_w`, `actual_power_source`, `is_consuming`, `battery_soc` і `battery_soc_blocked`.

## Температурна компенсація

Ця функція дозволяє коригувати розрахунок максимальної потужності (MPP) залежно від температури навколишнього середовища — ефективність панелей змінюється з температурою.

- **Сенсор температури** — сенсор температури навколишнього середовища в °C.
- **Температурний коефіцієнт Voc (%/°C)** — температурний коефіцієнт напруги холостого ходу (Voc) з datasheet панелі. Зазвичай від'ємне значення.
- **Температурний коефіцієнт Pmax (%/°C)** — температурний коефіцієнт максимальної потужності (Pmax) з datasheet панелі. Також зазвичай від'ємне значення.

## Розширені налаштування

Цей розділ дозволяє тонко налаштувати поведінку алгоритму розподілу потужності.

- **Резерв акумулятора (Вт)** — потужність, зарезервована для заряду акумулятора. Ця кількість не розподіляється між пристроями. Корисно для забезпечення мінімального заряду батареї.
- **Власне споживання інвертора (Вт)** — потужність, яку споживає сам інвертор. Віднімається від доступної сонячної потужності для точнішого розрахунку реального надлишку. Значення можна знайти в datasheet інвертора або виміряти.
- **Стратегія розподілу потужності** — визначає спосіб розподілу між пропорційними пристроями:
  - **Заповнювати по одному (Fill one by one)** — пристрій з найвищим пріоритетом отримує стільки, скільки потрібно; залишок іде до наступного.
  - **Розподіляти рівномірно (Distribute evenly)** — доступна потужність ділиться між активними пропорційними пристроями пропорційно до їх `Макс. очікуваної потужності`.
- **Мінімальна напруга інвертора** — мінімальна напруга, необхідна для роботи інвертора.
- **Крок збільшення (%)** — відсоток збільшення потужності для пропорційних пристроїв за кожен цикл.
- **Крок зменшення (%)** — відсоток зменшення потужності за кожен цикл.
- **Мертва зона рампи (%)** — невеликий діапазон навколо цільової потужності, у якому зміни не вносяться. Запобігає мікроколиванням.
- **Гістерезис (Вт)** — буфер потужності для запобігання частим перемиканням. Пристрій вмикається при досягненні мінімальної потужності і вимикається при `Мінімальна потужність − Гістерезис`.
- **Стартовий захисний період (с)** — час у секундах після ввімкнення пристрою, протягом якого він не буде вимкнений, навіть якщо потужність впаде нижче порогу. Дає пристроям час вийти на робочий режим.
