# Інтеграція ESPHome

SunAllocator може безпосередньо керувати кількома пристроями ESPHome для використання надлишкової сонячної енергії. Ця інтеграція дозволяє автоматично регулювати потужність твердотільних реле залежно від доступного невикористаного потенціалу з пріоритетним розподілом потужності.

## Підтримка кількох пристроїв

SunAllocator підтримує налаштування та керування кількома пристроями ESPHome. Ключові можливості:

- **Додавання кількох пристроїв**: Налаштуйте будь-яку кількість пристроїв ESPHome для використання надлишкової сонячної енергії.
- **Пріоритетний розподіл потужності**: Призначайте пріоритети пристроям для контролю того, які з них отримують живлення першими.
- **Індивідуальне налаштування пристроїв**: Кожен пристрій має власні налаштування для сутності реле, режиму, автокерування тощо.
- **Централізоване управління**: Керуйте всіма пристроями через єдиний інтерфейс конфігурації.

## Налаштування

При налаштуванні інтеграції SunAllocator спочатку конфігуруються параметри сонячних панелей, після чого можна додавати та керувати пристроями ESPHome:

1.  **Налаштування сонячних панелей**:
    *   Налаштуйте сенсори сонячних панелей, параметри напруги/струму тощо. Дивіться [Детальне налаштування](./configuration_uk.md).

2.  **Керування пристроями**:
    *   Додавайте, редагуйте або видаляйте пристрої ESPHome.
    *   Для кожного пристрою налаштуйте:
        *   **Назва**: Описова назва пристрою.
        *   **Тип пристрою**: Оберіть між «Без пристрою», «Стандартний перемикач/освітлення» або «Кастомне реле ESPHome».
        *   **Сутність реле ESPHome** (необов'язково): Сутність освітлення, що керує твердотільним реле.
        *   **Сутність вибору режиму ESPHome** (необов'язково): Сутність select, що керує режимом роботи.
        *   **Автокерування увімкнено**: Чи повинен SunAllocator автоматично керувати цим пристроєм.
        *   **Мінімальне споживання (Вт)**: Мінімальне корисне навантаження пристрою. Нижче цього порогу пристрій залишається вимкненим (з урахуванням гістерезису).
        *   **Максимальне споживання (Вт)**: Фізичний/логічний максимум пристрою; 100% у пропорційному режимі відповідає цьому значенню, і виділення обмежується ним.
        *   **Пріоритет**: Значення від 1 до 100, що визначає, які пристрої отримують живлення першими (вище — вищий пріоритет).
        *   **Розклад увімкнено**: Чи вмикати керування за часовим розкладом для цього пристрою.
        *   **Час початку**: Час, коли пристрій повинен починати роботу (якщо розклад увімкнено).
        *   **Час завершення**: Час, коли пристрій повинен припиняти роботу (якщо розклад увімкнено).
        *   **Дні тижня**: Дні, в які пристрій повинен працювати (якщо розклад увімкнено).

> **Примітка**: Як сутність реле ESPHome, так і сутність вибору режиму ESPHome є необов'язковими. Це дозволяє створювати пристрої, що використовують лише одну з цих сутностей або жодну. Наприклад, можна створити пристрій, що використовує лише сутність реле для прямого керування потужністю, або пристрій, що використовує лише сутність вибору режиму для керування зовнішньою системою.

## Типи пристроїв

SunAllocator підтримує три типи пристроїв:

1.  **Без пристрою (Placeholder)**: Запис-заповнювач без реальних сутностей керування. Корисний для планування або тестування.

2.  **Стандартний перемикач/освітлення (тільки увімкн./вимкн.)**: Стандартна сутність перемикача або освітлення Home Assistant, що підтримує лише режим увімкнення/вимкнення. Ці пристрої не використовують сутність вибору режиму і керуються безпосередньо на основі доступної надлишкової потужності.

3.  **Кастомне реле ESPHome (Увімкн./Вимкн./Пропорційний)**: Кастомний пристрій ESPHome з сутністю реле та сутністю вибору режиму, що підтримує всі три режими роботи (Вимкнено, Увімкнено, Пропорційний).

## Режими роботи

Кожен пристрій підтримує три режими роботи:

1.  **Вимкнено (Off)**: Реле повністю вимкнено.
2.  **Увімкнено (On)**: Реле увімкнено на повну потужність.
3.  **Пропорційний (Proportional)**: Потужність реле регулюється пропорційно до доступної надлишкової потужності.

## Розклад

Для кожного пристрою можна налаштувати розклад, що визначає, коли він повинен працювати:

1.  **Розклад увімкнено**: Перемикач для увімкнення або вимкнення розкладу для пристрою.
2.  **Час початку**: Час, коли пристрій повинен починати роботу (наприклад, «08:00»).
3.  **Час завершення**: Час, коли пристрій повинен припиняти роботу (наприклад, «20:00»).
4.  **Дні тижня**: Оберіть дні, в які повинен працювати пристрій.

Якщо розклад увімкнено, пристрій працюватиме лише в межах зазначеного часового проміжку у вибрані дні. Поза розкладом пристрій буде автоматично вимкнено.

Функція розкладу підтримує нічні розклади (коли час завершення раніше часу початку). Наприклад, якщо встановити час початку «22:00» і час завершення «06:00», пристрій працюватиме з 22:00 до 06:00 наступного дня.

> **Примітка**: Розклад застосовується після автокерування. Якщо автокерування вимкнено, пристрій не буде керуватися автоматично незалежно від розкладу.

## Служби

SunAllocator надає дві служби для керування пристроями ESPHome:

#### `sun_allocator.set_relay_mode`

Встановлює режим роботи одного або кількох реле.

Параметри:

-   `entity_id` (необов'язково): Ідентифікатор сутності конкретної сутності вибору режиму.
-   `device_id` (необов'язково): Ідентифікатор конкретного пристрою для керування.
-   `mode`: Режим роботи для встановлення. Повинен бути одним із: `Off`, `On`, `Proportional`.

Якщо ні `entity_id`, ні `device_id` не вказано, режим буде встановлено для всіх налаштованих пристроїв.

#### `sun_allocator.set_relay_power`

Встановлює рівень потужності одного або кількох реле.

Параметри:

-   `entity_id` (необов'язково): Ідентифікатор сутності конкретної сутності реле.
-   `device_id` (необов'язково): Ідентифікатор конкретного пристрою для керування.
-   `power`: Рівень потужності для встановлення у відсотках (0–100).

Якщо ні `entity_id`, ні `device_id` не вказано, потужність буде встановлено для всіх налаштованих пристроїв.

Дивіться [Приклади](./examples_uk.md) для прикладів використання.

## Пріоритетний розподіл потужності

Якщо налаштовано кілька пристроїв з увімкненим автокеруванням, SunAllocator розподіляє доступну надлишкову потужність на основі пріоритетів пристроїв:

1.  Пристрої сортуються за пріоритетом (спочатку вищий пріоритет).
2.  Пристрій з найвищим пріоритетом отримує живлення першим.
3.  Якщо після задоволення потреб пристрою з найвищим пріоритетом залишається потужність, вона переходить до наступного пристрою.
4.  Це продовжується до тих пір, поки вся надлишкова потужність не буде розподілена або всі пристрої не будуть задоволені.

Це дозволяє створити ієрархію навантажень. Наприклад:

-   **Пріоритет 100**: Критичні навантаження (наприклад, заряд акумулятора).
-   **Пріоритет 75**: Важливі навантаження (наприклад, нагрів води).
-   **Пріоритет 50**: Корисні навантаження (наприклад, обігрів приміщень).
-   **Пріоритет 25**: Необов'язкові навантаження (наприклад, підігрів басейну).

## Автоматичне керування

Якщо автокерування увімкнено (варіант A), SunAllocator регулює кожен пристрій з урахуванням меж очікуваного навантаження та гістерезису:

1.  Ефективний поріг увімкнення: пристрій стає активним, коли доступний надлишок перевищує `max(min_expected_w, Default Min Start (W))` з гістерезисом. Вмикається вище `+H/2` і вимикається нижче `−H/2` від цього порогу.
2.  Пропорційні пристрої (Custom, режим = Proportional): відсоток цільової потужності лінійно масштабується до можливостей пристрою: `target% = clamp(5..90, 100 × available_excess / max_expected_w)`. Виділена потужність у ватах обмежується значенням `max_expected_w`.
3.  Пристрої увімкн./вимкн. (Standard або режим = On): пристрій ВМИКАЄТЬСЯ коли активний, ВИМИКАЄТЬСЯ — інакше. Виділення обмежується `max_expected_w` (якщо задано), або невеликим внутрішнім резервним обмеженням. Фактичний відсоток відображається як 100% при ввімкненому стані, 0% при вимкненому.

## Поведінка з необов'язковими сутностями

Інтеграція обробляє пристрої з відсутніми сутностями наступним чином:

-   **Якщо у пристрою не налаштовано сутність реле**:
    -   Пристрій не отримуватиме команди керування потужністю.
    -   Служби, що націлені на цей пристрій для керування потужністю, занесуть попередження до журналу, але продовжать роботу для інших пристроїв.
    -   Автокерування пропустить цей пристрій при розподілі потужності.

-   **Якщо у пристрою не налаштовано сутність вибору режиму**:
    -   Пристрій не отримуватиме команди зміни режиму.
    -   Служби, що націлені на цей пристрій для зміни режиму, занесуть попередження до журналу, але продовжать роботу для інших пристроїв.
    -   Автокерування пропустить цей пристрій, оскільки не зможе визначити поточний режим.

-   **Якщо у пристрою не налаштовано жодної сутності**:
    -   Пристрій буде фактично вимкнено.
    -   Він відображатиметься в списку пристроїв, але не братиме участі в жодних операціях керування.

Така гнучкість дозволяє створювати пристрої-заповнювачі або пристрої, що використовують лише один аспект системи керування.

## Код компонента ESPHome

Для використання цієї інтеграції необхідні пристрої ESPHome з твердотільними реле та компонентами вибору режиму. Можна використати надану конфігурацію `sun_allocator_relay.yaml` як відправну точку:

```yaml
# Basic ESPHome configuration
esphome:
  name: sun_allocator_relay
  platform: ESP8266
  board: d1_mini

# WiFi connection
wifi:
  ssid: !secret wifi_ssid
  password: !secret wifi_password

# Enable Home Assistant API
api:
  encryption:
    key: !secret api_encryption_key

# Define the solid-state relay output
output:
  - platform: esp8266_pwm
    id: relay_output
    pin: D1
    frequency: 1000Hz

# Define a custom PWM light to control the relay
light:
  - platform: monochromatic
    name: "Sun Allocator Relay"
    output: relay_output
    id: relay_light
    restore_mode: ALWAYS_OFF

# Mode selection
select:
  - platform: template
    name: "Relay Mode"
    id: relay_mode
    options:
      - "Off"
      - "On"
      - "Proportional"
    initial_option: "Off"
    optimistic: true
```
