# SunAllocator 1.1.1

## Highlights

- Reworked allocator-triggered service calls to use non-blocking dispatch paths for entity control, watchdog shutdown, and auto-control turn-off flows.
- Hardened dispatcher and debounce rerun handling against recursive re-entry on current Home Assistant releases.
- Completed the missing advanced-settings translation labels in English and Ukrainian and aligned the English configuration docs with the UI wording.
- Fixed the device-removal options-flow path so deleted devices are cleaned up without double-removing their entities on current Home Assistant releases.

## Behavioral Changes

- `sensor.sun_allocator_power_distribution` now defers dispatcher-triggered state writes to the next loop tick and coalesces repeated updates before flushing state to Home Assistant.
- Overdue debounce rechecks are now scheduled one loop tick later instead of creating an immediate allocator task in the same stack.
- Service helpers now dispatch `blocking=False` calls so allocator control paths do not wait synchronously for Home Assistant service completion.
- Removing a configured device now reloads the integration first and then prunes the corresponding Home Assistant device-registry entry, avoiding duplicate entity removal during the transition.
