# SunAllocator 1.1.0

## Highlights

- Added optional dual-MPPT support with independent PV2 inputs and per-MPPT panel overrides.
- Added per-device actual-power, active-feedback, and minimum battery SOC controls for smarter handling of standard loads.
- Added partial allocator recompute and serialized reruns so device, feedback, and battery updates do not race each other.
- Added richer diagnostics, including the `idle` device status and expanded per-device sensor attributes.
- Updated English and Ukrainian documentation to match the new runtime behavior and configuration options.

## Behavioral Changes

- `sensor.sun_allocator_excess_power` now reports the full available solar headroom when a consumption sensor is configured. It is no longer limited to untapped power above the current operating point.
- Watchdog freshness now follows confirmed excess-sensor updates only, while auxiliary relay and feedback changes trigger partial recomputes without masking a stale solar signal.
- Disabling a device's auto-control switch can now optionally send an immediate `off` command to the managed entity.
